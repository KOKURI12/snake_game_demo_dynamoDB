# AGENTS.md

## 共通ルール

このプロジェクトでは、以下のルールを守って作業してください。

---

## 回答ルール

- 回答・説明は日本語で行うこと。
- 作業前に、まず関連ファイルを確認すること。
- コード修正前に、変更方針を簡潔に説明すること。
- 依頼内容に必要な最小限の修正だけを行うこと。
- 無関係なリファクタリングは行わないこと。
- 既存機能を壊さないことを優先すること。

---

## セキュリティルール

以下のファイル・情報を読まない、表示しない、変更しないこと。

- `.env` / `.env.*`
- AWS 認証情報（Access Key、Secret Key、Session Token）
- API キー / シークレットキー / token
- `*.pem` / `*.key`
- `credentials` 系ファイル・ディレクトリ
- `secrets/` ディレクトリ配下のファイル

---

## 禁止操作

以下の操作は事前にユーザー確認を得ずに実行しないこと。

- 新しい依存パッケージの追加（npm install / pip install 等）
- DB の削除・テーブル変更（DROP / DELETE / TRUNCATE 等）
- AWS リソースの変更・削除
- IAM 権限の変更
- `git push`
- `git reset --hard`
- `rm -rf`
- 本番環境（prd）へのデプロイ・変更
- 外部 API への送信・書き込み

---

## 推奨作業フロー

1. `PROJECT_CONTEXT.md` を確認する
2. 関連ファイルを確認する
3. 変更方針をユーザーに説明する
4. ユーザーの承認を得てから最小限の修正を行う
5. プロジェクトに応じた確認コマンドを実行する
6. 変更内容・確認結果・懸念点をまとめて報告する

---

## 確認コマンド（プロジェクトに存在する場合）

プロジェクトの `PROJECT_CONTEXT.md` に記載のコマンドを使用すること。
代表的な確認コマンド例：

```bash
# Node.js 系
npm run lint
npm test
npm run build

# Python 系
pytest
flake8 .

# 共通
git diff
git status
```
